# Reactr Template

A React + Vite template built around **feature-based architecture** and **automatic route generation**.

The template includes the **Reactr CLI**, which manages:

* feature creation
* component scaffolding
* router updates
* configuration updates

The goal is to keep project structure predictable while eliminating most manual boilerplate.

This template assumes **CSS Modules for component styling** and keeps **global styles isolated under `/src/styles`**.

---

# Core Ideas

### Features drive routing

Every feature corresponds to a **route in the application**.

The CLI creates the feature folder, updates the configuration, and regenerates the router automatically.

Example:

```
reactr add dashboard
```

Creates:

```
src/features/dashboard
 ├─ DashboardShell.tsx
 ├─ DashboardShell.module.css
 └─ pages
     ├─ DashboardPage.tsx
     └─ DashboardPage.module.css
```

And automatically updates the router so the route becomes:

```
/dashboard
```

---

### Router is generated automatically

Routes are derived from `reactr.config.ts`.

Example config:

```ts
export const reactorConfig = {
  rootFeature: "home",
  features: [
    "home",
    "dashboard",
    "settings"
  ]
};
```

Produces routes:

```
/           → HomeShell
/dashboard  → DashboardShell
/settings   → SettingsShell
```

The **first feature added becomes the root route** unless another root is explicitly set.

---

# Component Generator

The CLI can also generate **components inside a feature**.

Example:

```
reactr add component Button dashboard
```

Creates:

```
src/features/dashboard/components/Button
 ├─ Button.tsx
 ├─ Button.module.css
 └─ index.ts
```

Generated files include:

**Component**

```tsx
export function Button() {
  return <div className={styles.root}>Button</div>;
}
```

**CSS Module**

```
.root {

}
```

**Barrel export**

```ts
export * from './Button';
```

This allows components to be imported cleanly from the feature.

Example:

```ts
import { Button } from '../components/Button';
```

# Styling

The template assumes **CSS Modules for component styles**.

Each component receives its own module file:

```
Component.tsx
Component.module.css
```

Example:

```tsx
import styles from './Button.module.css';
```

### Global styles

Global styles are isolated in:

```
src/styles
```

These are intended for:

* resets
* tokens
* global typography
* CSS variables
* theme setup

Component styles should remain local through CSS modules.

# Project Structure

```
src
 ├─ app
 │   ├─ App.tsx
 │   └─ Router.tsx
 │
 ├─ features
 │   └─ (generated features live here)
 │
 ├─ shared
 │   └─ reusable UI components
 │
 ├─ styles
 │   └─ global styles
 │
 └─ reactr.config.ts
```

### `features`

Contains route-based application features generated by the CLI.

### `shared`

Reusable components not tied to a single feature.

Examples:

* layout primitives
* navigation
* shared UI elements

### `styles`

Global styling such as resets and tokens.

### `reactr.config.ts`

Configuration file used by the CLI to generate routes.

# Reactr CLI

The CLI is responsible for maintaining project structure.

---

### Add a feature

```
reactr add featureName
```

Creates the feature folder, registers it in the config, and regenerates the router.

---

### Delete a feature

```
reactr delete featureName
```

Removes the feature folder, removes it from the config, and updates the router.

---

### Add a component

```
reactr add component ComponentName featureName
```

Creates a component inside the feature and generates:

* component file
* CSS module
* barrel export

# New Project Checklist

After creating a new project from this template:

* Rename the project in `package.json`
* Update the `<title>` in `index.html`
* Set `BASE_URL` in `.env` if deploying under a subpath
